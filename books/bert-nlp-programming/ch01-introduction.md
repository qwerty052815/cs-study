# Topic
第一章　はじめに

## 1. Core Idea
- **自然言語処理(NLP)**：自然言語の関わる問題(タスク)をコンピューターで解くこと
- **機械学習**：大量のデータを用いた、学習と推論の2つの過程を通じて、何らかの問題を解くような方法。一般に高性能なモデルの構築には大量のデータが必要になる場合が多く、データ収集や計算コストが課題となることがある。
- **深層学習**：end-to-end形態を採用した学習方法
**表現学習**：有用な特徴を学習することを目的とした学習方法
既存の機械学習は特徴量抽出方法(人間)にモデル(AI)の性能が依存する問題があったが、深層学習や表現学習では、データから特徴量を自動的に学習できるため、従来手法と比べて多くのタスクで性能向上が報告されている。
- **ニューラル言語モデル**：文章や単語を「密なベクトル(分散表現)」として変換可能である。NLPの多くのタスクでは分類問題として定式化されることが多く、文章間または単語間の類似性をベクトル空間上で定量的に評価できため、有用性が高い言語モデルである。
- **Bert**：文脈を考慮した分散表現（Attention）が可能であるニューラル言語モデル。学習方法として、事前学習とファインチューニングの過程があることが特徴的である。

## 2. Definitions
- **自然言語**＝日常生活で人間が用いる言葉
**人工言語**＝特定の用途のためだけに作られた言葉
- **教師あり学習**＝ラベル付きデータを扱う機械学習の一種
- **学習**＝データを用いて、望ましい入出力関係をモデルに学習させること
- **推論**＝学習済みモデルを用いて、入力でたのラベル(正解)を予測すること
- **end-to-end**＝特徴量抽出とその後のモデルによる処理が一つのモデルで完結すること
- **事前学習**＝大量の文章データを用いて、汎用的言語のパターン学習させること
- **ファインチューニング**＝比較的少数のラベル付きデータを用いて、特定のタスクに特化するように学習させること

## 3. Mechanism
- NLPの技術
1. 形態素解析
2. 固有表現抽出
3. 文章の類似度比較
4. 文章分類
5. 文章生成
6. 文章校正

- 機械学習の問題解法
1. データからタスクを解くのに有用な特徴量を抽出
2. 抽出しら特徴量をモデルに入力、出力から問題解く

- (学習)前処置
1. モデルに入力するために、データを数値に変換
2. モデルの性能向上のために、データの「重要な変数を選択」「統計量を計算」「次元を圧縮」など何らかの処理を行う

## 4. Hypothesis
- 教師あり学習でラベル付きデータを使うということは、条件付けを通じて、ラベルごとに文章分類可能なため、もしかするとNLPの技術でも特に文章分類に強みを示す可能性があるのではないか？
- 表現学習が有用な特徴をデータから学習するということは、もしかすると固有表現抽出に強みを示す可能性があるのではないか？

## 5. AI Feedback
- 分類問題として定式化できるNLPタスクとラベル付きデータを用いる教師あり学習は相性が良い。(仮説の原因と結果が反対)
- 表現学習はデータから有用な特徴表現を学習するため、固有表現抽出を含む多くのNLPタスクに有効である可能性がある。(仮説の一般化が必要)

## 6. Open Questions
- NLPが上記で説明した6つの技術以外の技術、もしくは拡張性を持っている可能性は？
- 機械学習でない学習方法を採用するAIと機械学習AIの間に関わりはないのか？
- 自然言語という文字を具体的にどうやって数値で表せることが出来るのか？
- 重要な変数の”重要”とはどの基準によって区別されているのか？データの共通点/相違点を基準に区別する場合も存在するのか？
- 次元圧縮とはどういう意味か？
- 深層学習や表現学習の場合、モデルが何を基準に特徴量を抽出し、何を根拠に抽出された特徴量が適応的だと言えるのか？
- ニューラル言語モデルでない言語モデルがデータをベクトルに表せないということならば、モデルのどこからそのような相違点が出現されるのか？根本的に、ニューラル言語モデルは何を基準に言語という文字をベクトルで変換することが出来るのか？ここで言うベクトルとは、空間で方向を持ったベクトルでなく、スカラー倍と四則演算が可能な意味合いでのベクトルと予想されるが、だとすれば、どのような原理に基づいて自然言語をベクトルとして表現できるのか？
- “Bertが大量のデータを事前に学習することにより、何からの意味で言語を「理解」できるようになり、その結果として個別のタスクでは比較的少数のデータから問題を解けるようになる”と説明されているが、ここでいう「理解」とは出力から逆算した意味合いで理解したと言うのか？もしくは体系的一般化が可能であることを意味するのか？
