第二章　ニューラルネットワークを用いた自然言語処理

## 1. Core Idea
[前処理]
- **トークン化**：文章を適切な単位に分割すること
- ニューラルネットワークを構築する際、表れる未知語を無視するか、特定のIDにマッピングすることから情報が失われる問題がある。しかし、未知語を可能な限り減らすために、語彙数を可能な限り増やすと逆に計算量も増えるため、未知語への対応と計算量のバランスが取れた方法でトークン化を行うことが必要
- **単語分割**：文章を単語に分割するトークン化方法の一種。トレードオフの関係（未知語と計算量）をはらんでいる。日本語のように単語の区切りが明確でない言語は形態素解析ツールを利用する。（MeCab、Sudachi、Jumanなど）
- **文字分割**：文章を1文字ごとに分割するトークン化方法の一種。語彙が文字数と同等であるため、語彙は非常に小さくなるが、トークン数は大幅に増加し、それに応じて計算量も増加する。また、語彙が具体的な意味と紐付かないため、ニューラルネットワークは文字の系列が持つ意味を考慮する必要があり、さらに、単語は一文字の違いで意味が大きく変容するという課題がある。
- **サブワード分割(Bert採用)**：文章を単語に、単語をサブワードに分割するトークン化方法の一種。計算量と未知語への対応のバランスが取れたトークン化が可能なため、未知語に強い性質がある。サブワード分割を効果的なものにするためには、語彙を適切に作成する必要がある。そのため、BPEやWordPieceなどを利用する。
- Ex) “私は東京タワーにいる”→ “私/は/東京タワー/に/いる”(形態素) “東京タワー”→”東京”, “##タワー”

[ニューラル言語モデル]
- **ニューラル言語モデル**：ニューラルネットワークにより実現される言語モデル。ニューラル言語モデルを学習する過程で得られるトークンの分散表現は、タスク非依存の汎用的な言語的特徴（トークンの意味や品詞などの統計的情報）を表すとされ、その有用性はさまざまなタスクへ応用を通じて示されている。
- **言語モデル**：文の生起確率をモデル化する。具体的には、人間がもっともらしい（ありえる）と感じる文章には高い確率を、そうでない文章には低い確率を与える。文章の自然さを確率によって表現しているため、音声認識や機械翻訳、文章誤り訂正など自然な文章を扱いたい領域で広く応用される。ある文脈下で出現するトークンを予測するモデルとも言える。
- **ニューラルネットワーク**：神経回路網をモデル化する試みに起源を持つ数理モデル。なんらかの変換を行う層（layer）の組み合わせによって構成される。各層は一般に、入力値に対して、適当な線形変換と活性化関数が施された出力値を返すが、近年では、単純な線形変換と非線形変換を複数組み合わせて作られる、より複雑な処理単位を層と呼ぶことも一般的である。複数の層を組み合わせることにより高い表現能力を示す。各層で行われた変換は、パラメータに依存し、これらのパラメータが学習の過程で調整されることで、ニューラルネットワークは適切に問題を解くことができるようになる。一方、層の組み合わせや各層における入出力値の次元数などは学習過程で変化しないため、ハイパーパラメータに分類される。パラメータを調整する際、現在のパラメータの良さを示す定量的な基準が必要である。この良さとは、そのパラメータが持つニューラルネットワークが出現する入出力関係とタスクで獲得したい理想的な入出力関係を比較することで確認でき、ズレが少なければ少ないほど良いパラメータであると言えるため、損失関数を用いて、損失を最小化するようにパラメータを調整する。
- **分類問題**：NLPの多くのタスクは与えられたデータに対して、与えられた複数のカテゴリーの中からそのデータに対応するカテゴリーを決める分類問題として扱うことができる。出力がカテゴリーと同じサイズのベクトルになるようにニューラルネットワークを設計することにより、分類問題を解けるようになる。
- **分類スコア**：出力ベクトルyの各要素値のそれぞれカテゴリーへの予測確度。分類スコアが最も高いカテゴリーがニューラルネットワークの予測である。

[ニューラル言語モデルの構築]
- **入力層**：文脈に該当するトークン列を入力として受け取る層である
- **埋め込み層(Embedding Layer)**：トークンを意味のある入力とするため、それぞれを密な（すべての次元に意味のある値が入る）ベクトル表現に変換する層である。N x Dの行列Eによって特徴付けられる。埋め込み層で得られる分散表現が、学習により適切に決まる。
(N = 語彙の大きさ, D = 分散表現として割り当てたいベクトルの次元数)
- **出力層**：与えられた文脈下でのトークン出現確率分布を出力する層である。語彙数Nに対応する次元のベクトルを出力するようにし、モデルの出力の次元がカテゴリーの数と異なる場合には、線形変換（もしくはより複雑な変換）によりN次元のベクトルに変換したものを出力とみなす。その後、出力にSoftmax関数を適用し、語彙中のそれぞれのトークンの確率を得る。なお、損失関数についても多クラス分類問題と同様にクロスエントロピー誤差を用いる。
- **ニューラルネットワークの学習データ**：自然言語で記述された任意の文章データ（コーパス）とトークナイザを用いることで自動的に作成可能である。モデル学習のため、大量に用意し「文脈 - 正解」の組である。
- **ニューラルネットワークの分散表現**：ニューラル言語モデルが推論の過程で中間出力するトークンの分散表現はトークンの言語的特徴を捉える＝人間が似ていると思うトークンは似たような分散表現を持つ
- **分布仮説**：語の意味はその周辺に出現する語によって表される
- **言語モデルの課題**：似たような文脈で出現する対義語関係(Ex)暑い/寒い)、そもそも文章に出現しにくい自明な常識(Ex)バナナは黄色い)のような言語的特徴を捉えることを苦手とする
- **分散表現の類似性**：分散表現の類似性を表す定量的な尺度として一般にコサイン類似度が用いられる。

[ニューラル言語モデルの例1 Word2Vec]
- **Word2Vec**：トークンに対して文脈に依存せず一意な表現を与える、単語分割型のニューラル言語モデルの一種。加法構成性を持つため、計算効率に優れ、従来難しいとされてきた数GB単位のコーパスを用いる数十万単位の語いの学習も可能である。GensimなどのPythonライブラリーを提供しているため、気軽に動作を試みることも可能である。CBOWとskip-gramの2つのモデルが提案されている。
- **CBOW(Continuous Bag-Of-Words)**：文章S = (w1,w2,…,wn)が与えられたとき、i番目に位置する単語wiをその周りの単語(文脈)Ci = (wi-c ,…, wi-1, wi+1 ,…, wi+c,)から予測する言語モデル。文脈中の単語をベクトルに変換する埋め込み層Vc = vc(w)と予測する単語をベクトルに変換する埋め込み層Vt = vt(w)で構成されている。確率分布p(wi | Ci)をモデル化する。巨大な語彙を扱う学習の際に、計算量がボトルネックとなる問題を対処するため、Hierarchical SoftmaxとNoise Contrastive Estimation(NCE)のいずれかを用いて計算量削減し、大規模な語彙の学習を可能にする。学習済みCBOWから単語の分散表現を得るために、埋め込み層Vcを用いる。
- **skip-gram**：文章中の単語wiが与えられたとき、文脈Ciに出現するwの確率分布p(w | wi)をモデル化する（CBOWの因果関係を逆転させたようなモデルであるため、確率計算における考え方やモデル構造が類似）
- ** Word2Vecの問題点**：各単語に対して一意に分散表現を与える性質は多義語を扱う際に問題がある。2つ以上の意味を持つ単語に対しての分散表現が一意に定まるため、文脈に応じた意味の変化を扱うことができない。また、単語の分散表現の重み付き平均を利用することで、良い性能が得られるが、語順が考慮されない。
Ex) 彼は舞台の上手に立った。彼は料理上手だ。ジョンはボブに本を貸した＝ボブはジョンに本を貸した(分散表現一致)

[ニューラル言語モデルの例2 ELMo]
- **ELMo**：トークンに対して文脈に応じた表現を与える単語分割型のニューラル言語モデルの一種。出力する文脈を捉えた単語の分散表現を特徴量として用いることは、さまざまなタスクにおいて有用性がっ示されている。文脈に応じた単語の分散表現を得るため、再帰型ニューラルネットワークの一種であるLSTMを用いる。
- **RNN**：再帰型ニューラルネットワークの最も単純なモデル。入力された情報を長期間保持できない短所がある。
- **LSTM(Long-Short Term Memory)**：RNNの構造上の問題を解決するために提案されたモデル(大域的な処理の流れは類似)各時刻で2つの内部状態(hi, ci = メモリセル)を持ち、外に出力されるのはhiのみで、ciは内部でのみ受け渡しが発生する。(RNNと同様の入出力関係を持つモジュールとして取り扱い可能)内部に1つのRNNと情報の流れを制御する3つゲート（入力、忘却、出力）が存在、各ゲートは対応する入出力値に対する重み（その入出力値を考慮/忘却する割合）を計算する。算出される重みがLSTMへの入力に応じて変化することで、柔軟な時系列情報の考慮が可能となる。

## 2. Definitions
[前処理]
- **トークン**＝トークン化によって得られた文章の構成要素
- **トークナイザー**＝トークン化を実現するツール
- **語彙**＝事前に適当な方法で入力として扱いたいトークンの集合
- **未知語**＝入力として受け入れることが出来る語彙は固定されるため表れる、語彙に含まれないトークン
- ** 形態素解析ツール**＝文章を分割するだけでなく、得られた形態素に対し品詞や活用における基本形などの統計的情報を付与する。注釈付きコーパスを利用して構築されている。
- ** コーパス**＝話し言葉を書き起こした文章や記事などの文章を大量に収集し、検索や分析ができるようにしたデータベース
- **注釈付きコーパス**＝分割境界や品詞タグなどの情報を付加したコーパス
- **サブワード**＝部分文字列
- **Byte Pair Encoding(BPE)、WordPiece**＝サブワード分割のためにコーパスから語彙を作成するアルゴリズム。文章を分割したときのトークン数が平均的に小さくなる等に、指定されたサイズの語彙を自動的に生成する。

[ニューラル言語モデル] - **活性化関数**＝入力に対して非線形変換を適用する関数。Sigmoid、ReLU、GELUなどあり。
- **パラメータ**＝各層が持つ調整可能な変数
- ** ハイパーパラメーター**＝人手を要する設定事項
- **ラベル**＝カテゴリーを0からN-1の整数で表現したもの

[ニューラル言語モデルの構築]
- **埋め込み表現(Embedding) = 分散表現**：トークンに対応付けられたベクトル。言語的特徴が特徴空間に埋め込まれている(ベクトルの各次元に分散して現れる)ことの期待に由来している。
- **多クラス分類**＝ある文脈下で、語彙中のN個の選択肢の中から現れるトークンを予測する問題はカテゴリーの数をNとする多クラス分類として解釈可能
-**CommonCrawl**＝多くのサイトからさまざまな文書データが収集されているデータセット

[ニューラル言語モデルの例1 Word2Vec]
- **単語埋め込み（Word Embedding）**＝トークンに文脈非依存の分散表現を与えること
- **加法構成性**＝数値の加減算と意味的な加減算が一致するような性質
Ex) v(日本) - v(東京) ≒ v(フランス) - v(パリ), v(日本) + v(首都) ≒ v(東京)
- **ウインドウサイズ**＝CBOWで文脈の範囲の広さを制御できるハイパーパラメータc

[ニューラル言語モデルの例2 ELMo]
- **文脈化単語埋め込み(Contextualized Word Embedding)**＝トークンに文脈に応じた分散表現を付与すること。
- ** 再帰型ニューラルネットワーク**＝系列データを処理するためのモデル。
- **順方向RNN**＝前方から処理行うRNN
- **逆方向RNN**＝後方から処理行うRNN
- **多層RNN**＝複数層を積み上げて処理行うRNN
- **多層LSTM**＝複数層を積み上げて処理行うLSTM
- **双方向LSTM(Bidirectional LSTM)**＝順方向の多層LSTMと逆方向のLSTMを組み合わせたLSTM＝主なELMoの構成

## 4. Formula
- 文章Sの出現確率(w1,w2,w3… = トークン, ci = 文脈(context))
p(S) = p(w1,w2,w3…) = p(w1) x p(w2 | w1) x p(w3 | w1,w2)… = n Π i=1 p(wi | ci)
したがって、文章全体の出現確率は、各時点における条件付き確率の積としてモデル化できる。

- 分類器
分類問題のゴールはデータxが与えられたときにそのラベルlをよく観測できるモデル
- 出力
y = F(x,θ) (カテゴリーの数と同じ次元を持つベクトル)
(N = カテゴリー数, x = データを表現するベクトル, l = ラベル, θ = 調整可能なパラメータ）

- 各カテゴリーの予測確率(SoftMax関数)
pi = exp(yi)/Σ N j=1 exp(yj) (i=1,2,…N)
ここで、0<pi<1であり、p1+p2+…+pn=1を満たすため、(p1,p2…pn)を確率分布として扱える。

- 損失関数
ズレを定量的に表現する、パラメータの関数 L(y,l) L = (1/m) Σ m i=1 L(yi, li), L(y,l) ‎ =  -log pi(クロスエントロピー損失）

- コサイン類似度（正規化されたベクトル同士の内積）
cos(x,y) = x・y / |x| |y|

- CBOWの文脈
v(Ci) = 1/2c Σ wj∈Ci vc(wj)
- 文脈Ciにおいて、語彙中のj番目のトークンsjの出現しやすさ
v(Ci)・vt(sj)
- Softmax関数で確率に変換した結果(語彙数Nに比例) p(wi = sj | Ci) = exp{v(Ci)・vt(sj)}/Σ N k=1 exp{v(Ci)・vt(sk)}

-skip-Gramの確率分布
p(sj|wi) = exp{vc(wi)・vt(sj)}/ΣN k=1 exp{vc(wi)・vt(sk)}

- RNNの出力
hi = φ(Axi+Bhi-1+b) (i=1,2,…,n)
(hi = RNNの出力, xi = 同時刻の入力, hi-1 = 前時刻のRNNの出力, A/B = 行列, b = ベクトル, φ = 活性化関数, h0 = 初期値 = 適当に決めたベクトル）
φ(Axi+Bhi-1+b) (i=1,2,…,n) ‎ =  φ(xi,hi-1)仮定、
hi = φ(xi,hi-1) = φ(xi, φ(xi-1,hi-2)) = φ(xi, φ(xi-1, φ(xi-2,hi-3)))…
RNNは現在時刻以前に行われた入力{xi}考慮して、出力を計算可能

- 言語モデルとしてのRNN
hi = hi-1(w1,w2,…,wi-1)
p(wi=sj | w1,w2,…,wi-1) = exp(h’i-1,j)/Σ N k=1 exp(h’ i-1,k)
~hi = φ(~Axi + ~B~hi+1 + ~b) (i=1,2,…,n)
hi^(1) = φ(A^(1)xi+B^(1)h^(1)i-1+b^(1)) hi^(k) = φ(A^(k)hi^(k-1)+B^(k)h^(k)i-1+b^(k))
(入力xi = 単語wiの分散表現, h’i-1 = すべての語彙に出現確率を割り当てるため、出力に適当な線形変換を用いて、語彙と同じサイズにしたベクトル, ベクトルh’i-1のj番目の要素h’i-1,j)

- LSTMの出力
yi = φ(Axi+Bhi-1+b) (xi = 現在時刻の入力, yi = 出力)
- LSTMの重み
gI = σ(AIxi+BIhi-1+bI)
gF = σ(AFxi+BFhi-1+bF)
gO = σ(AOxi+BOhi-1+bO)
ci = gI◉yi+gF◉ci-1 hi = gO◉tanh(ci)
(xi = 現在時刻の入力, hi-1 = 1つ前の時刻の出力, gI(Input Gate),gF(Forget Gate),gO(Output Gate) = 重みベクトル, σ = 活性化関数(要素ごとにシグモイド関数を適用し、重みベクトルの各要素の値域は(0,1)に抑えられる), ◉ = 要素ごとの積)

- 言語モデルとしてのELMo
順方向の多層LSTM(文章の前の文章と文脈)と逆方向の多層LSTM(文章の後の文章と文脈)からの予測確率を組み合わせて、損失関数
- Σ n i=1 log p(wi | w1, w2,…,wi-1) + log p(wi | wi+1, wi+2,…,wn)
を用いて学習する。予測対象以外の全てを文脈として考慮していると言える。

- 特徴量抽出器としてのELMo ElMoを用いてi番目の単語wiの特徴量を抽出することを想定した場合、順方向LSTMの出力hi^(k)と逆方向LSTMの出力~hi^(k)を結合し
